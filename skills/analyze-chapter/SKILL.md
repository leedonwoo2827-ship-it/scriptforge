# 챕터 분석 + 씬 선별

챕터 원고를 읽고, knowledge/scene-design-guide.md의 6대 기준에 따라 씬 후보를 식별하고, 서사적 아크를 고려하여 최적의 씬 라인업을 제안한다.

## 사전 조건

- 챕터 MD 파일이 `sample/` 폴더에 있어야 한다
- `output/_series_guide.json`이 있으면 장르/씬 수/스타일을 자동 참조
- 시리즈 가이드가 없으면 사용자에게 장르를 직접 물어본다

## 분석 절차

### Step 1: 메타데이터 추출

챕터 MD 파일의 첫 부분에서:
- `> 제N부: 제목 (시대)` — 부(Part) 정보
- `# N장. 제목` — 장 번호, 제목
- `_부제목_` — 핵심 메시지

### Step 2: 섹션 구조 파싱

`##` 헤더 기준으로 섹션을 분리한다:
1. `## 참고문헌` 이전: **본문** (씬 추출 대상)
2. `## 참고문헌`: 참고 (씬 추출 대상 아님)
3. `# 인물부록` 이후: 부록 (인물 정보 확인용만)

각 본문 섹션에 대해:
- 섹션 제목
- 글자 수 (분량 파악)
- 1-2줄 요약

### Step 3: 씬 후보 식별

각 본문 섹션을 읽으며 scene-design-guide.md의 6대 기준을 적용:

1. **시간적 앵커**: 구체적 연도/날짜가 등장하는 문장 식별
2. **인물 행동**: "이름 + 동사" 패턴 식별 (예: "스키너는 설계도를 그렸다")
3. **물리적 공간**: 구체적 장소 묘사 식별
4. **극적 긴장감**: 감정/갈등/전환점 표현 식별
5. **시각적 고유성**: 다른 후보와 시각적으로 구분되는지 판단
6. **서사적 역할**: 기(opening)/승(development)/전(climax)/결(closing) 중 어디에 해당

### Step 4: 씬 라인업 제안

후보 중 시리즈 가이드의 `scenes_per_episode` 범위(기본 5-8개)에 맞게 선별:

- **필수 포함**: 시간적 앵커 + 인물 행동이 동시에 있는 장면
- **서사 아크**: 기승전결 구조가 되도록 배치
- **시각적 다양성**: 같은 구도/톤의 씬이 연속되지 않게

### Step 5: 라인업 보고

사용자에게 다음 형식으로 제시:

```
📋 챕터 분석: N장. [제목]
   부: 제N부 [부제목] | 본문 섹션: M개 | 씬 후보: K개

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎬 추천 씬 라인업 (N개)

 1. [씬 가제목]
    출처: ## [섹션명]
    핵심: [1줄 내용 요약]
    서사 역할: [기/승/전/결]
    예상 시간: [N초]

 2. [씬 가제목]
    ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 총 예상 시간: N분 N초
   제외된 주요 후보: [간략히]
   수정하시려면 씬 추가/삭제/순서변경을 말씀해주세요.
```

## 사용자 수정 대응

- "씬 추가해줘: [설명]" → 해당 섹션에서 추가 후보 식별
- "3번 빼줘" → 라인업에서 제거, 서사 아크 재확인
- "순서 바꿔: 2→4" → 재배치
- "씬 좀 더 넣어줘" → 목표 씬 수 상향, 추가 후보 제안
- "줄여줘" → 가장 약한 씬 제거 제안

확정되면 다음 단계(prompt-craft)로 진행.

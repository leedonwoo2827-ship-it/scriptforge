# /forge-review

기존 대본을 수정한다.

## 실행 흐름

### Step 1: 기존 대본 로드

- 챕터 번호로 지정: `output/ch{NN}/ch{NN}_script.json` 로드
- 파일 경로로 직접 지정도 가능
- 현재 씬 라인업을 요약 표시

```
📋 3장 현재 대본: 5씬 | 2분 0초 | classic-documentary

 1. 1953년 하버드 교실 (25초)
 2. 스키너의 분노 (20초)
 3. 그날 밤의 설계도 (22초)
 4. 잊혀진 선구자 (28초)
 5. 프레시의 자동 교사 (25초)

수정할 내용을 말씀해주세요.
```

### Step 2: 사용자 수정 반영

가능한 수정 사항:

#### 씬 구조 변경
- **씬 추가**: "스키너의 티칭 머신 씬 추가해줘" → 원고에서 해당 장면 찾아 새 씬 생성
- **씬 삭제**: "3번 빼줘" → 삭제 후 번호 재정렬
- **씬 순서 변경**: "4번을 2번 뒤로" → 재배치
- **씬 분할**: "1번을 두 씬으로 나눠줘" → 분할

#### 프롬프트 수정
- "1번 프롬프트를 좀 더 극적으로" → 프롬프트 재작성
- "전체적으로 톤 밝게" → 모든 씬 프롬프트의 톤 조정
- "모델 전부 imagen으로 바꿔" → 일괄 모델 변경

#### 내레이션 수정
- "2번 내레이션 수정: [새 내용]" → 내레이션 교체, 시간 재계산
- "전체 내레이션 톤 바꿔줘" → 모든 내레이션 재작성
- "시간 줄여줘" → 내레이션 압축

#### 메타데이터 수정
- "자막 추가/변경" → scene_meta.subtitle 수정
- "BGM 힌트 변경" → scene_meta.bgm_hint 수정
- "전환 효과 변경" → scene_meta.transition_hint 수정
- "오프닝/엔딩 텍스트 변경" → video_meta 수정

### Step 3: 변경사항 미리보기

수정된 부분만 Before/After로 표시:

```
🔄 변경사항:

Scene 3 (수정됨):
  제목: 그날 밤의 설계도
  프롬프트: "A study desk scene..." → "A dimly lit study room at night..."
  내레이션: 변경 없음
  시간: 22초 → 22초

Scene 6 (추가):
  제목: 스키너의 티칭 머신
  프롬프트: "A mechanical teaching device..."
  내레이션: "1954년, 스키너는 최초의 티칭 머신을..."
  시간: 24초

전체: 5씬 → 6씬 | 2분 0초 → 2분 24초

적용할까요?
```

### Step 4: 파일 재생성

사용자 확인 후:
- JSON 덮어쓰기
- MD 재생성
- CSV 재생성
- 매니페스트 업데이트 (수정 시간, 씬 수 등)

```
✅ 3장 대본 업데이트 완료

변경: 씬 1개 추가, 프롬프트 1개 수정
총: 6씬, 2분 24초
```

## 주의사항

- FlowGenie에서 이미 이미지를 생성한 씬의 프롬프트를 수정하면, 이미지를 재생성해야 한다
- 씬 번호가 바뀌면 image_filename도 자동 업데이트된다
- 원본 JSON은 덮어쓰기 전에 기존 내용을 보고한다 (복구 가능하도록)

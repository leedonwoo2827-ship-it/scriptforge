# /forge

챕터 원고에서 마스터 대본을 생성하는 전체 플로우.

## 실행 흐름

### Step 1: 입력 확인 + 설정

- 챕터 MD 파일 경로 확인 (sample/ 폴더)
- `output/_series_guide.json` 존재 여부 확인
  - **있으면**: 장르, 씬 수, 색상 톤, 내레이션 스타일 자동 로드. 해당 챕터의 에피소드 정보 표시
  - **없으면**: 사용자에게 장르 선택 요청 (기본: classic-documentary)
- 설정 요약 보고: 장르, 목표 씬 수, 예상 영상 길이

**사용자 확인 후 다음 단계. "바꿔줘"라고 하면 설정 변경.**

### Step 2: 챕터 분석 + 씬 라인업 (analyze-chapter)

analyze-chapter 스킬을 실행한다:
- 챕터 구조 분석 (본문 섹션 파싱)
- 씬 후보 식별 (6대 기준)
- 추천 씬 라인업 제안 (서사 아크 고려)

**라인업을 사용자에게 제시. 확인/수정 대기.**

가능한 사용자 응답:
- "좋아" / "확인" → Step 3으로
- "3번 빼고 다른 거 넣어줘" → 라인업 수정 후 재제시
- "씬 2개 더 추가해줘" → 추가 후보 제안
- "순서 바꿔" → 재배치

### Step 3: 씬별 상세 설계 (prompt-craft)

prompt-craft 스킬을 실행한다:
- 2-3씬씩 묶어서 보여주기
- 각 씬: 이미지 프롬프트 + 내레이션 + scene_meta + 파일명

**2-3씬 보여주고 확인 → 다음 2-3씬 → ... → 전체 완료.**

가능한 사용자 응답:
- "좋아" → 다음 씬 묶음으로
- "1번 프롬프트 수정: [내용]" → 수정 후 재제시
- "내레이션 톤 바꿔줘" → 전체 톤 조정
- "전부 한번에 보여줘" → 나머지 한꺼번에 표시

### Step 4: 출력 생성 (generate-script)

generate-script 스킬을 실행한다:
- JSON 파일 저장 (FlowGenie/TTS/SceneWeaver용)
- MD 대본 저장 (사람이 읽는 대본)
- CSV 저장 (Excel 검토용)
- 매니페스트 업데이트
- 결과 요약 보고

**완료.**

## 진행 원칙

1. **각 단계마다 사용자 확인** — 한 번에 전체를 생성하지 않는다
2. **되돌아가기 가능** — "다시 분석해줘" → Step 2로
3. **시리즈 일관성** — 시리즈 가이드가 있으면 색상 톤, 내레이션 스타일, 모델 등을 가이드에 맞춘다
4. **정보 부족 시 질문** — 가정하지 않고 사용자에게 확인

## 최종 산출물

```
output/ch{NN}/
├── ch{NN}_script.json    ← 마스터 대본 (FlowGenie/TTS/SceneWeaver)
├── ch{NN}_script.md      ← 사람이 읽는 대본
└── ch{NN}_script.csv     ← Excel 검토용
```

## 연속 실행

한 챕터 완료 후:
- "다음 챕터도 해줘" → 다음 장 번호의 MD 파일로 /forge 재실행
- "나머지 전부" → /forge-batch로 전환 제안
